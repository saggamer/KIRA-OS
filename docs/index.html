Built as a local agentic environment, not a chatbot skin.

Orchestrator chooses the route. KIRA OS provides the computer pathways, permission checks, generated artifacts, live context, and proof layer.

01 / context

Computer awareness

Read-only scans for apps, files, windows, RAM, storage, downloads, project state, and live computer context.

02 / create

Artifact generation

Fresh PPTX decks with images, DOCX reports, PDFs, notes, package manifests, and research summaries.

03 / act

Desktop pathways

Open apps, browse the web, inspect pages, connect IDEs, draft MCP configs, and prepare automation steps.

04 / guard

Safety boundary

Read-first behavior, permission-gated mutation, macOS sandbox direction, protected-path blocking, and failure reporting.

05 / verify

Evidence layer

The agent should return file paths, browser state, command results, screenshots, or logs before claiming completion.

06 / scale

Council and background work

Confidence-based council mode, background tasks, live mode, and multi-step recovery for hard agentic workflows.
